CDMS-Agent Harness Engineering

CRF/DVS 테스트 자동화는 과제마다 UI와 규칙이 크게 달라집니다. 현재 하네스의 중심은 데이터를 장기간 축적해 추천하는 방식이 아니라, CRF 코드와 현재 브라우저 상태를 함께 보고 지금 실행 가능한 검증 셀만 만드는 browser-assisted 방식입니다.

현재 방향

Daemon Inspect 확장

inspect()가 단순 visible_rows뿐 아니라 row별 editable, disabled, controlType, options를 반환합니다.

자동 생성 셀 보호

생성된 notebook 셀은 입력 전에 row_is_editable()로 현재 UI 상태를 확인하고, 비활성 row는 건너뜁니다.

누적 데이터 제거

SQLite raw log와 pending suggestion 흐름은 현재 active harness에서 제외합니다. 승인된 override JSON만 유지합니다.

처리 흐름

CRF TypeScript source
  -> extract_spec / CRFRunner
  -> Query candidate cases
  -> current browser inspect
  -> structuredRows row-state guard
  -> generated notebook cells
  -> step-by-step run
  -> save
  -> post-save query check

이 흐름은 “모든 DVS rule을 완벽히 코드로 변환”하는 목표가 아닙니다. 우선 현재 페이지에서 실제로 누를 수 있는 입력만 수행하고, 저장 후 query 발생 여부를 사람이 확인하기 쉬운 notebook 셀로 나눕니다.

하네스 계층

계층목적검증 기준
Doctor harness 브라우저 실행 전 CRF 소스 품질 확인 duplicate trigger id, unresolved itemId, missing pageId, query/availability trigger 요약
Inspect harness 현재 브라우저 UI 상태 수집 structuredRows에 row label, editable, disabled, controlType, options 포함
Generation harness 실행 가능한 notebook 셀만 생성 현재 page/path와 맞는 case, row-state guard 포함, Python syntax 통과
Browser smoke harness 실제 CDMS 화면에서 단계별 검증 입력, save, post-save query check가 셀 단위로 확인 가능

Inspect 데이터 모델

PageSnapshot
  visible_rows: list[str]
  structured_rows: list[dict]

structured_rows item
  rowLabel: string
  visible: boolean
  editable: boolean
  disabled: boolean
  controlType: "radio" | "checkbox" | "textarea" | "select" | "input" | "unknown"
  options: string[]

예를 들어 가임 여부에서 예를 선택한 뒤 비가임 사유가 비활성화되면, notebook은 row_is_editable("비가임 사유") 결과를 보고 해당 입력을 건너뛰고 다음 row로 진행합니다.

cd C:\Users\SunbeomGwon\CDMS-Agent

개발 프롬프트 기준

Claude/Codex에게 CRF 자동화 수정을 맡길 때는 다음 조건을 명시합니다.

작업은 browser-assisted harness 기준으로 진행한다.

요구사항:
- SQLite raw accumulation / SuggestionEngine 기반 추천 기능은 active path에 추가하지 않는다.
- daemon inspect가 structuredRows(rowLabel, editable, disabled, controlType, options)를 반환하게 한다.
- notebook 생성 셀은 입력 전에 row_is_editable(row label)을 확인한다.
- 비활성 row는 실패로 보지 않고 SKIP 출력 후 다음 step으로 넘어간다.
- 저장 후 query 확인은 post-save snapshot 기준으로 한다.
- 기존 gen_notebook, doctor 동작을 깨뜨리지 않는다.

완료 기준:
- cdms-crf doctor가 실행된다.
- 생성 notebook에 row_is_editable guard가 포함된다.
- pytest가 통과한다.
- 문서에는 SQL 누적/추천 흐름이 active 기능처럼 설명되지 않는다.

제외한 기능

  • 실행 실패 raw log를 SQLite에 장기 축적하는 흐름
  • 누적 로그 기반 pending suggestion 생성/승인 CLI
  • LLM이 과제별 지식을 자동으로 확정해 override JSON에 쓰는 흐름

이 기능들은 나중에 재검토할 수 있지만, 현재 목표인 “되는 것만 자동 실행하고 멈추지 않기”에는 브라우저 현재 상태를 직접 읽는 하네스가 더 단순하고 안정적입니다.
