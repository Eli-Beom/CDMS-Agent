CDMS-Agent Daemon Connection Guide

이 문서는 CDMS-Agent daemon을 실행하고, Chrome Extension 및 Python CDMSAgent가 daemon을 통해 실제 CDMS UI를 제어하는 연결 구조를 설명합니다. 현재 기본 daemon 주소는 http://127.0.0.1:3200입니다.

빠른 실행

루트에서 실행

cd C:\Users\SunbeomGwon\CDMS-Agent
npm run daemon

daemon 폴더에서 실행

cd C:\Users\SunbeomGwon\CDMS-Agent\daemon
npm run dev

두 명령은 같은 개발 서버를 실행합니다. 루트의 npm run daemon은 내부적으로 npm run dev --prefix daemon을 호출합니다.

프로세스 시각화

1. Daemon 실행 npm run daemon으로 Express 서버를 :3200에서 시작
2. Extension 연결 Chrome Extension service worker가 daemon WebSocket에 접속
3. Browser Runner 주입 CDMS 페이지에 runner core가 주입되고 현재 탭이 client로 등록
4. Python 명령 Notebook의 CDMSAgent가 daemon HTTP API 호출
5. UI 실행/결과 daemon이 WebSocket으로 브라우저에 명령을 보내고 결과를 Python에 반환

Socket과 Extension 연결 구조

Python / Notebook

CDMSAgent가 HTTP 요청을 보냅니다.

  • agent.ping()
  • agent.inspect()
  • agent.set_date(...)
  • agent.select_radio(...)
  • agent.click_save()

Daemon

HTTP API와 WebSocket bridge를 동시에 제공합니다.

  • HTTP: /api/cdm-agent/*
  • WebSocket: /ws/cdm-agent
  • 기본 포트: 3200

Chrome Extension

daemon과 socket으로 연결되고 CDMS 탭에서 runner를 실행합니다.

  • service-worker.js
  • browser-runner-core.js
  • CDMS DOM 조작 및 snapshot 반환
Python -> HTTP -> Daemon
http://127.0.0.1:3200
Daemon <-> WebSocket <-> Extension
ws://127.0.0.1:3200/ws/cdm-agent

연결 순서

1. daemon 실행 PowerShell에서 npm run daemon을 실행하고 listening on http://127.0.0.1:3200 로그를 확인합니다.
2. Extension 로드 Chrome 확장 관리 페이지에서 C:\Users\SunbeomGwon\CDMS-Agent\extension 폴더를 unpacked extension으로 로드합니다.
3. CDMS 페이지 열기 지원 대상 호스트는 sbx.cdms.mavenclinical.com 및 cdms.mavenclinical.com입니다.
4. Extension 상태 확인 side panel에서 daemon origin이 http://127.0.0.1:3200인지 확인하고 reconnect합니다.
5. Python 연결 확인 Notebook에서 agent.ping()과 agent.inspect()를 실행합니다.

Browser-Assisted 생성에서의 역할

browser-assisted notebook 생성은 daemon 연결을 실행 단계뿐 아니라 생성 단계에서도 사용합니다. 생성기는 agent.clients()로 현재 Extension client를 고정하고, agent.inspect()로 현재 page, pathname, visible rows를 읽은 뒤 CRF 후보 케이스를 필터링합니다. 

daemon running
+ extension client registered
+ CDMS page open
+ agent.inspect()
= current browser context for script generation

따라서 과제를 변경한 뒤에는 브라우저에서 원하는 subject/visit/page를 먼저 열고 notebook을 생성합니다. 이때 생성된 notebook은 해당 화면 상태에 맞는 자동화 가능 케이스만 포함합니다.

Python 연결 확인

from cdm_agent_client import CDMSAgent

agent = CDMSAgent(study_id="PRACTICE_GSB")

print("daemon:", agent.ping())
snapshot = agent.inspect()
print("page:", snapshot.page_label)
print("path:", snapshot.pathname)

CDMSAgent의 기본 base_url은 현재 http://127.0.0.1:3200입니다. 별도로 지정하려면 CDMSAgent(base_url="http://127.0.0.1:3200", ...)처럼 넘길 수 있습니다.

주요 포트와 파일

항목 현재 값 파일
daemon 기본 포트 3200 daemon/src/index.ts
Python client 기본 URL http://127.0.0.1:3200 src/cdm_agent_client/client.py
Extension daemon origin http://127.0.0.1:3200 extension/service-worker.js, extension/sidepanel.html
WebSocket endpoint ws://127.0.0.1:3200/ws/cdm-agent daemon/src/index.ts, extension/service-worker.js

문제 확인

daemon unreachable

npm run daemon이 실행 중인지, 포트가 3200인지 확인합니다.

browser client 없음

CDMS 탭을 열고 extension side panel에서 reconnect합니다.

UI 명령 실패

현재 페이지가 예상 visit/page인지, row label이 실제 화면과 일치하는지 확인합니다.

요약

CDMS-Agent는 Python이 브라우저 DOM을 직접 만지는 구조가 아닙니다. Python은 daemon에 HTTP 요청을 보내고, daemon은 WebSocket으로 연결된 Extension에 실행 명령을 전달합니다. Extension은 CDMS 페이지 안에서 runner를 실행해 실제 입력, 저장, query 확인을 수행한 뒤 결과를 daemon과 Python에 돌려줍니다.
