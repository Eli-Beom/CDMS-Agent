CDASH/CDISC 표준 룰 테이블

패턴추천 타입근거
*DTCDATECDASH: Date/Time Character (ISO 8601)
*DATDATECDASH: Date field
*CATDROPDOWNCDASH: Category (코드화된 값)
*SCATDROPDOWNCDASH: Subcategory
*YNRADIOCDASH: Yes/No indicator
*FLRADIOCDASH: Flag (Y/N)
*ONGORADIOCDASH: Ongoing indicator
*ORRESTEXTSDTM: Original Result (자유 텍스트)
*STRESCTEXTSDTM: Standardized Result (character)
SEXRADIOCDASH: Sex (M/F/U)
RACEDROPDOWNCDASH: Race (코드 목록)
ETHNICDROPDOWNCDASH: Ethnicity
COUNTRYDROPDOWNCDASH: Country (ISO 3166)

v2.0 — Extension 서버 연동 설계 (planned)

HTML (브라우저) ↓ fetch('http://localhost:3200/apply-change', { itemId, pageId, field, value }) crf-reviewer dev server (Express, :3200) ↓ patch .ts file (@babel/parser AST 수정) maven-crfs/src/crfs/<study>/page.ts ↓ npx tsx generate-preview.ts 재생성 preview-*.html → browser auto-reload

기존 CDMS daemon과의 차이

CDMS daemon(:3100)은 EDC 브라우저 조작용 WebSocket 서버.
CRF reviewer 서버(:3200)는 로컬 파일 편집 전용.
역할이 달라 별도 프로세스로 운영.

AST 수정 전략

간단한 변경(layout, appearance)은 regex로 충분.
복잡한 변경(format 객체 교체)은 @babel/parser AST 수정 권장.
maven-crfs는 이미 @babel/parser 의존성 보유.
