Rule Discovery 결과판정 로직 수정 보고서

작성일: 2026-05-29 / 대상: CDMS-Agent browser-assisted query discovery

문제 정의

Discovery loop에서 입력 step이 실패했는데도 observed_query_count = 0이라는 이유만으로 Result = no_query_observed가 기록되는 문제가 있었다. 또한 query_observed는 화면에 Query가 하나라도 있는지 기준이어서, 의도한 DVS rule의 Query인지와 다른 row의 누락 Query인지가 구분되지 않았다.

수정 내용

항목수정 전수정 후
step error 처리 입력 실패 후 query가 없으면 no_query_observed 입력 step 중 하나라도 실패하면 blocked
예상-관측 관계 Expected Result와 Result를 사용자가 직접 비교 discovery_category로 관계를 명시
대상 Query 식별 전체 Query 메시지만 출력 target_observed_query_count, target_observed_query_messages 추가
출력 표 query_observed, no_query_observed, manual_review_or_unknown 예상 일치/불일치/blocked 표를 추가 출력

새 판정 기준

조건Resultdiscovery_category
Save 실패 또는 입력 step 실패blockedblocked_or_skipped
runner skipped 발생skippedblocked_or_skipped
query 예상, 실제 query 관측query_observedexpected_and_observed
query 예상, 실제 query 미관측no_query_observedexpected_but_not_observed
no query 예상, 실제 query 관측query_observednot_expected_but_observed
no query 예상, 실제 query 미관측no_query_observednot_expected_and_not_observed

주의점

observed_query_count는 현재 화면 전체에서 관측된 Query 수다. 따라서 다른 필수 입력 누락 Query가 섞일 수 있다. 의도한 DVS rule과 직접 관련된 메시지는 target_observed_query_messages를 우선 확인한다.
