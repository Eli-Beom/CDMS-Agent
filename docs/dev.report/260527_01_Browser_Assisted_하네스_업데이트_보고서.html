Browser-Assisted Harness Update Report

Project: CDMS-Agent Date: 2026-05-27 (_260527) Scope: CRF/DVS notebook automation

이번 업데이트의 핵심 의도는 “CRF 코드를 완벽히 해석해서 모든 DVS rule을 자동화”하는 방향이 아니라, 현재 브라우저 화면을 inspect해서 지금 실제로 입력 가능한 항목만 안전하게 실행하도록 하네스를 바꾸는 것입니다.

개발 배경

  • 과제별 CRF 구조, label, option, precondition이 크게 달라 정적 코드 분석만으로는 안정적인 자동화가 어렵습니다.
  • 예: 가임 여부 = 예 선택 후 비가임 사유가 비활성화되면, 기존 셀은 해당 row를 계속 누르려다 멈출 수 있습니다.
  • Query는 입력 직후가 아니라 저장 후 표시되므로, step 실행과 save 이후 확인을 분리해야 합니다.
  • SQLite에 실패 데이터를 계속 축적해 추천하는 방식은 현재 단계에서는 운영 부담이 커서 active 기능에서 제외했습니다.

변경된 방향

1. CRF 코드Query 후보 case 생성
2. Browser Inspect현재 화면 row 상태 확인
3. Row Guard입력 가능 row만 실행
4. Save저장 후 피드백 대기
5. Query Checkpost-save snapshot 확인

기본 모드는 browser-assisted입니다. CRFRunner가 만든 후보를 현재 브라우저 상태와 결합하고, notebook 셀은 사람이 한 단계씩 확인하며 실행할 수 있게 생성됩니다.

Inspect로 확인하려는 것

단순히 화면의 텍스트를 모으는 것이 아니라, row 단위로 다음 정보를 분리해서 봅니다.

  • rowLabel: 자동화가 조작할 기준 row 이름. 예: 성별, 가임 여부, 비가임 사유, 수유 여부
  • options: radio/select 선택지. 예: 남성, 여성, 예, 아니요
  • editable: 지금 입력 가능한지 여부
  • disabled: 보이지만 비활성화되어 있는지 여부
  • controlType: radio, input, textarea, select 등 입력 방식

현재 상태와 다음 작업

현재 extension 연결은 정상이며, structuredRows가 daemon을 통해 Python까지 전달됩니다. 다만 row label과 option 분리가 완벽하지 않은 화면이 있을 수 있어, 실제 DM 화면에서 row 추출 품질을 계속 보정해야 합니다.
