목표

이번 수정의 목표는 DVS/query rule 검증이 아니라, 연결된 브라우저의 현재 페이지에서 실제 입력 가능한 row를 더 정확히 확정하고 browser-assisted notebook에 입력 명령을 생성하는 것이다. query expected result, has_query, check_result, CASE_CHECK는 계속 제외했다.

주요 변경

structuredRows 보강 visible text 후보가 아니라 실제 control을 기준으로 row 후보를 잡도록 변경했다. 가까운 tr을 우선 row container로 사용하고, 없으면 wrapper/section/parent 계층을 사용한다.
rowLabel 정리 option 단독 텍스트인 예, 아니요, 남성, 여성, 기타 등이 rowLabel로 확정되지 않도록 필터링했다. bracket label인 [좌측 유방] 수술 목적, [우측 유방] 수술 목적은 보존했다.
row 상태 필드 structured row에 visible, editable, disabled, row_visibility, row_availability, row_disability를 포함하도록 했다.
radio 클릭 범위 제한 getRadioLabelCoords(rowLabel, optionLabel)가 document 전체가 아니라 findRow(rowLabel) 내부에서만 option을 찾도록 수정했다. 예/아니요 중복 row에서 다른 row를 클릭하는 위험을 줄였다.
CDP 클릭 검증 service worker가 CDP로 radio를 클릭한 뒤 isRadioOptionSelected()로 checked 상태를 확인한다. 검증 실패 시 noop으로 넘기지 않고 runner의 selectRadio fallback step을 유지한다.
notebook 생성 보강 _browser_input_cases()는 visible/editable row를 우선 사용하되, 생년월일, 비가임 사유처럼 runner skip/특수 입력이 필요한 row는 입력 step에 포함되도록 보정했다. _preferred_option()은 해당사항 없음, 아니요, 예 순서를 기본으로 하고, 가임 여부/성별/수유 여부/좌우측 수술 목적/유방암 병력은 명시 우선값을 사용한다.
