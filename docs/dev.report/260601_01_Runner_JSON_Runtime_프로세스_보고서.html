candidate JSON의 의미

candidate JSON은 runner 자체가 아니라, runner가 만든 실행 후보 데이터다. 노트북 셀에 긴 후보 목록을 직접 넣지 않고 별도 파일로 분리해 가독성과 재사용성을 높인다. 

{
  "DVS ID": "D_DM_1_2",
  "item_id": "BRTHDAT",
  "item_label": "생년월일",
  "rule_type": "range_query",
  "rule_source": "CRF+Browser",
  "Specification": "AGE<22",
  "Expected Result": "query_expected",
  "steps": [
    {
      "method": "set_date",
      "args": ["생년월일", "2005-01-01"],
      "kwargs": {"page_id": "DM", "visit_id": "V1"}
    }
  ]
}

CSV의 의미

CSV는 candidate가 아니라 실행 결과다. 노트북의 rule_discovery_result는 커널이 꺼지면 사라질 수 있으므로, 실행 후 결과를 파일로 남긴다.

주요 열은 DVS ID, Specification, Test Script, Expected Result, observed_query_count, observed_query_messages, target_observed_query_messages, Result, discovery_category, review_decision이다.

pytest의 의미

pytest는 실제 EDC 브라우저 클릭 검증이 아니라 Python 패키지 내부 로직 검증이다. 예를 들어 trigger 중복 탐지, override JSON load/save, discovery result 판정, step error의 blocked 처리 같은 로직이 깨지지 않았는지 확인한다.

정리

질문답
JSON이 runner인가?아니다. JSON은 runner가 만든 실행 후보 데이터다.
runner는 무엇인가?CRF 코드를 읽고 trigger/case 후보를 만드는 로직이다.
runtime은 무엇인가?candidate JSON을 읽어 브라우저에서 실행하고 결과를 누적하는 로직이다.
CSV는 왜 만드는가?노트북 실행 결과를 커널 밖에 보존하고 후속 검토/승인에 사용하기 위해 만든다.
pytest는 무엇인가?Python 코드 내부 로직이 깨지지 않았는지 확인하는 테스트 도구다.
