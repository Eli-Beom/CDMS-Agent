목적은 CRF query 후보를 Jupyter cell로 만들되, 사람이 브라우저 피드백을 보면서 한 단계씩 실행할 수 있게 하는 것입니다.

개발 의도

  • CDMS query는 저장 후 알 수 있으므로, 입력과 query 확인을 분리합니다.
  • 저장이 너무 빠르면 피드백을 못 보므로 PAUSE_BEFORE_SAVE, POST_SAVE_DELAY_SECONDS를 둡니다.
  • 현재 페이지가 지정 page와 다르면 빈 notebook을 만들지 않고, 후보를 유지한 뒤 실행 시 row guard로 보호합니다.
