Daemon vs Python 패키지 강화 평가

영역현재 문제와 역할우선순위
Daemon / Extension 실제 실패 지점은 날짜 입력창을 띄우지 못하거나 radio option을 클릭하지 못하는 DOM 조작 문제입니다. 이는 Python이 해결하기 어렵고, 브라우저 안에서 row/option/input을 더 정확히 찾아야 합니다. 우선 강화
Python CRF 패키지 Python은 CRF 후보 생성, 셀 순서 구성, guard 삽입, 실행 흐름 단순화에 집중하는 것이 맞습니다. DOM 클릭 성공률 자체를 Python에서 보정하려 하면 조건문과 예외가 과도하게 늘어납니다. 보조 강화

결론적으로 다음 작업의 중심은 daemon/extension입니다. Python 패키지는 “무엇을 시도할지”를 만들고, daemon/extension은 “실제 화면에서 어떻게 정확히 누를지”를 책임지는 구조로 나누는 것이 가장 실용적입니다.
