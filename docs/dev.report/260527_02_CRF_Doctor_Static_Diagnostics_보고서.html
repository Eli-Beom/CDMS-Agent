CRF Doctor / Static Diagnostics

Date: 2026-05-27 (_260527) · Work Unit: CRF source diagnosis

목적은 브라우저 실행 전에 CRF 소스의 구조적 문제와 자동화 가능 범위를 빠르게 확인하는 것입니다.

개발 의도

  • 중복 trigger id처럼 실행 전에 알 수 있는 오류를 먼저 차단합니다.
  • 모든 trigger를 자동화하지 않고, query auto와 manual review를 분리합니다.
  • 과제 규모를 숫자로 파악해 “어디까지 자동화할지” 판단 근거를 만듭니다.

구현 내용

항목내용
중복 ID 검사duplicated trigger id 탐지.
참조 검사unresolved itemId, missing pageId 확인.
Trigger 분류자동화 가능한 query와 EACH/CURRENT/SIBLING/dynamic context manual review 분리.
요약 출력page.ts, trigger.ts, query 합계, availability item 합계 출력.

검증 결과

cdms-crf doctor 실행 시 예시 과제에서 query trigger/auto/manual review/availability 합계와 warning/info가 출력됩니다.
