Prerequisite 최소 분리 및 Candidate Loop 재설계 계획 보고서

작성일: 2026-06-08

대상: CDMS-Agent Rule Discovery candidate 생성 및 notebook runtime loop

1. 배경

기존 계획은 모든 candidate에 prerequisite_category, validation_loop, referenced_item_ids, referenced_item_match_quality를 추가하는 방향이었다. 그러나 현재 문제는 세밀한 taxonomy가 아니라, setup이 필요한 query candidate와 일반 query candidate가 같은 loop에서 실행되며 입력 및 저장이 실패하는 것이다.

따라서 이번 단계에서는 과한 분류 체계를 만들지 않고, loop를 안전하게 나눌 수 있는 최소 flag만 추가한다.

2. 목표

candidate 생성 시점에 setup 필요 여부만 판정하고, notebook에서는 일반 query / setup 필요 query / availability 검증을 분리 실행한다.

3. 유지할 최소 metadata

Field필수 여부의미
requires_setup 필수 candidate 실행 전에 같은 page 내 control row 입력이 필요한지 여부
setup_steps setup 필요 시 필수 target/input row를 활성화하기 위해 먼저 실행 가능한 step 목록
setup_steps_count 권장 notebook 표에서 setup 필요 candidate를 빠르게 확인하기 위한 count
setup_reason 권장 예: same_page_availability
prerequisite_item_ids 선택 디버깅용 control item id 목록. 과한 taxonomy가 아니라 원인 확인용으로만 사용

4. 제거할 과한 분류

제거 대상제거 이유
prerequisite_category현재는 same-page setup 여부만 분리하면 충분함
validation_looprule_type과 requires_setup 조합으로 notebook에서 분리 가능
referenced_item_match_qualityrow resolver 개선 phase에서 다루는 것이 적절함
cross_page_prerequisiteDVS/cross-page dependency graph phase에서 별도 처리
unsupported_prerequisite, unknown_prerequisite이번 작업의 목적을 흐리므로 limitation/review note로만 남김

5.5 notebook loop 분리

수정 대상: C:\Users\SunbeomGwon\maven-crfs\scripts\generate_screening_rule_discovery.py

query_candidates = [
    c for c in DISCOVERY_CANDIDATES
    if c.get("rule_type") != "availability"
]

normal_query_candidates = [
    c for c in query_candidates
    if not c.get("requires_setup")
]

setup_required_query_candidates = [
    c for c in query_candidates
    if c.get("requires_setup")
]

availability_candidates = [
    c for c in DISCOVERY_CANDIDATES
    if c.get("rule_type") == "availability"
]
Loop실행 정책
일반 query loop normal_query_candidates만 실행
setup 필요 query loop setup_required_query_candidates를 표로 표시. 기본 자동 실행은 하지 않음
availability loop availability_candidates를 query loop와 분리해 실행

8. 실행 프롬프트

CDMS-Agent Rule Discovery candidate의 prerequisite 분리를 최소 변경으로 진행해줘.

목표:
세밀한 prerequisite taxonomy를 만들지 말고, setup이 필요한 query candidate와 일반 query candidate를 분리해서 notebook validation 정확도를 높인다.

수정 대상:
1. C:\Users\SunbeomGwon\CDMS-Agent\src\cdm_agent_client\crf\candidate_prerequisites.py
2. C:\Users\SunbeomGwon\maven-crfs\scripts\generate_screening_rule_discovery.py
3. C:\Users\SunbeomGwon\CDMS-Agent\src\cdm_agent_client\crf\rule_discovery.py
4. 필요 시 C:\Users\SunbeomGwon\CDMS-Agent\src\cdm_agent_client\crf\rule_discovery_runtime.py
5. tests/test_candidate_prerequisites.py

구현 요구사항:
- CRF page field의 availability를 스캔해서 candidate input item이 같은 page availability 조건을 갖는지 확인한다.
- 모든 candidate에 최소한 requires_setup boolean을 추가한다.
- setup이 필요한 candidate에는 setup_steps, setup_steps_count, setup_reason='same_page_availability', prerequisite_item_ids를 추가한다.
- prerequisite_category, validation_loop, referenced_item_match_quality 같은 과한 taxonomy는 추가하지 않는다.
- setup이 필요 없는 candidate에는 requires_setup=false만 보장한다.
- notebook에서는 candidate를 세 그룹으로 분리한다.
  1. normal_query_candidates: rule_type != 'availability' and requires_setup=false
  2. setup_required_query_candidates: rule_type != 'availability' and requires_setup=true
  3. availability_candidates: rule_type == 'availability'
- normal_query_candidates만 기본 query loop에서 실행한다.
- setup_required_query_candidates는 표로 표시하고, 기본 자동 실행하지 않는다.
- availability_candidates는 query loop와 분리한다.
- main query loop에서는 setup_steps를 자동 실행하지 않는다.
- Result 판정은 query 발생 여부만 보게 유지한다.

검증:

- 일반 query loop에 requires_setup=true candidate가 섞이지 않아야 한다.
- availability candidate가 query loop에 섞이지 않아야 한다.
