Screening Pages Rule Discovery 생성 보고서

개발 배경

DM 페이지에서 정리한 Rule Discovery notebook/candidate JSON/runtime loop 구조를 Screening visit 관련 CRF 페이지로 확장했다. 사용자가 적은 page id는 label과 CRF page 파일을 기준으로 재확정했다.

CDMS-Agent 구성요소와 작동 원리

CRFRunner가 TypeScript CRF page/trigger spec을 추출하고, candidate JSON은 자동화 가능한 입력 step만 보관한다. Notebook runtime은 load_candidates, run_rule_discovery_loop, display_rule_discovery_tables 순서로 실행되며 결과 표는 rule_discovery_result 하나만 표시한다. Result 값은 PASS 또는 FAIL이다.
