개발 의도

현재 단계의 목적은 DVS query 발생 여부를 자동 판정하는 것이 아니라, 연결된 브라우저의 현재 CRF 페이지에서 실제로 입력 가능한 항목을 최대한 입력하고 저장 전 피드백을 사람이 확인할 수 있게 하는 것이다. 따라서 Python notebook이 사전에 row_editable, row_locked 같은 helper로 분기하지 않고, 입력 명령은 그대로 보내며 실제 DOM 상태 판단은 extension runner가 담당하도록 정리했다.

Runner 변경

입력 계열 동작(setText, setDateViaCalendarPopup, selectRadio, selectComboboxOption) 실행 전에 target row 상태를 회사 trigger 용어 기준으로 판단하도록 했다.

row_visibilityrow가 화면에 표시되지 않으면 row_visibility_hidden으로 skipped 처리한다.
row_availability보이지만 현재 입력 가능한 control이 없으면 row_availability_unavailable로 skipped 처리한다.
row_disabilitydisabled, readOnly, aria-disabled 등으로 잠겨 있으면 row_disability_locked로 skipped 처리한다.
row_query현재 자동 판정 대상에서 제외했다.

row not found는 기존처럼 실패로 유지했다. 반대로 row가 보이고 입력 가능하다고 판단된 상태에서 option을 찾지 못하거나 click/input commit이 실패하면 진짜 실패로 남긴다.

Notebook 변경

생성 notebook에서 row 상태를 미리 판단하는 helper 함수와 query 자동 판정 문구를 제거했다. 셀은 agent.set_date(...), agent.select_radio(...) 같은 직접 입력 명령만 보여준다. 비가임 사유처럼 Availability 조건상 입력 불가인 항목은 notebook이 아니라 runner가 skipped로 처리하고 다음 step으로 넘어간다.

주의 사항

extension/service-worker와 browser-runner-core가 바뀌었으므로 Chrome 확장 프로그램은 새로고침 또는 재로드가 필요하다. 재로드 전에는 브라우저에 주입된 이전 runner가 남아 있어 row_availability_unavailable skip 결과가 보이지 않을 수 있다.

확장 프로그램을 재로드한 뒤 v0.3.7 notebook의 DM 입력 셀을 한 줄씩 실행한다. 특히 가임 여부=예 이후 비가임 사유가 입력 불가 상태일 때 cell이 멈추지 않고 runner result의 inputs에 outcome="skipped", reason="row_availability_unavailable" 또는 row_disability_locked가 남는지 확인한다.
