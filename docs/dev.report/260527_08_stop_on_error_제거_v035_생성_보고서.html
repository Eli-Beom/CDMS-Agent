stop_on_error 제거 및 v0.3.5 생성 보고서

Date: 2026-05-27 (_260527) Project: CDMS-Agent Notebook: CDMS-Agent_test_v0.3.5.ipynb

stop_on_error는 현재 daemon/extension 기반 실행 구조에서 실질적인 문제 해결 수단이 아니므로 제거했습니다. 이제 Python client와 생성 notebook은 실패를 예외 제어로 숨기지 않고, daemon이 반환하는 실행 결과를 그대로 다루는 방향입니다.

제거한 이유

항목문제결정
agent.stop_on_error = False notebook 셀에 반복적으로 들어가지만, 현재 실패 원인인 DOM 클릭/date commit 문제를 해결하지 못합니다. 생성 셀에서 제거.
CDMSAgent(stop_on_error=False) 사용자가 이 값을 바꿔도 daemon/extension runner의 실제 클릭 성공률에는 영향이 없습니다. Python client 생성자 옵션에서 제거.
CRFRun(..., stop_on_error=...) 하네스가 실패를 멈출지 말지 제어하는 것보다, 실패 원인을 DOM runner에서 바로 고치는 것이 우선입니다. CRF case 실행 wrapper에서 제거.

수정 내용

  • src/cdm_agent_client/client.py: stop_on_error 생성자 인자와 예외 분기 제거.
  • src/cdm_agent_client/crf/notebook.py: 생성 notebook에서 agent.stop_on_error = False 제거.
  • src/cdm_agent_client/crf/run.py, models.py: stop_on_error 전달 인자 제거.
  • docs, 기존 notebook 예제에서 stop_on_error 사용 흔적 제거.
  • CDMS-Agent_test_v0.3.5.ipynb 신규 생성.

결론

현재 자동화의 핵심 병목은 Python 예외 제어가 아니라 browser runner의 DOM 인식과 trusted click/date commit입니다. 따라서 stop_on_error는 제거하고, 문제는 daemon/extension runner에서 직접 고치는 방향이 맞습니다.
