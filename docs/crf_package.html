CDMS-Agent CRF Package Guide

src/cdm_agent_client/crf는 maven-crfs의 TypeScript CRF 소스를 읽고, 현재 브라우저 상태를 보조 정보로 사용해 실행 가능한 notebook 검증 셀을 생성합니다. 현재 기본 생성 방식은 browser-assisted입니다.

처리 흐름

TypeScript CRF
  -> extract_spec()
  -> CRFRunner
  -> CRFCase / Step / Check
  -> browser inspect PageSnapshot
  -> row-state guarded notebook cells

과제별 특이사항은 runner에 하드코딩하지 않고, 가능한 범위에서는 CRF 코드와 현재 브라우저 UI를 함께 봅니다. 장기 raw 데이터 축적이나 SQLite 기반 추천은 현재 active path에서 제외했습니다.

주요 모듈

파일역할주요 API
extractor.pyTypeScript CRF를 JSON spec으로 추출하고 field map을 만듭니다.extract_spec, build_field_map
parser.pytrigger와 UI rule을 자동화 가능/수동 검토 대상으로 분류합니다.classify_triggers
simulator.pyQuery 발생/미발생용 입력 후보를 계산합니다.extract_inputs
runner.pyCRF spec을 바탕으로 CRFCase 목록을 생성합니다.CRFRunner
notebook.pycase를 Jupyter Notebook 셀로 변환합니다. browser-assisted 모드에서는 row-state guard를 넣습니다.gen_notebook
doctor.py중복 trigger id, unresolved item/page, query/availability trigger 합계를 요약합니다.run_doctor
overrides.py사람이 승인한 과제별 label/option/precondition override만 JSON으로 관리합니다.load_overrides, save_overrides

Browser-Assisted 생성

입력 전 guard

생성 셀은 row_is_editable("row label")을 확인한 뒤 입력합니다. 비활성 row는 실패가 아니라 SKIP으로 넘깁니다.

현재 페이지 기준

이미 해당 페이지에 있으면 불필요한 go_to_page() 이동을 하지 않고 현재 화면에서 테스트합니다.

저장 후 query 확인

Query는 입력 즉시가 아니라 click_save() 후 snapshot 기준으로 확인합니다.
