CDMS-Agent Client Guide

cdm_agent_client는 Python/Jupyter에서 daemon HTTP API를 통해 Chrome extension runner를 호출합니다. Python이 DOM을 직접 조작하지 않고, 브라우저에 연결된 runner가 실제 CDMS 화면에서 입력과 저장을 수행합니다.

연결 구조

Notebook
  -> CDMSAgent
  -> daemon http://127.0.0.1:3200
  -> browser client websocket
  -> extension runner
  -> CDMS page
  -> StepResult / PageSnapshot

빠른 시작

import sys
sys.path.insert(0, r"C:\Users\SunbeomGwon\CDMS-Agent\src")

from cdm_agent_client import CDMSAgent

agent = CDMSAgent(
    base_url="http://127.0.0.1:3200",
)

print("daemon:", agent.ping())
snapshot = agent.inspect()
print("page:", snapshot.page_label)
print("path:", snapshot.pathname)
print("structured rows:", len(snapshot.structured_rows))

PageSnapshot

inspect()는 현재 페이지의 query 상태와 visible row뿐 아니라 row별 조작 가능 여부를 반환합니다. 이 값이 notebook의 row_is_editable() guard에 사용됩니다. 

snapshot = agent.inspect()

print(snapshot.page_label)
print(snapshot.pathname)
print(snapshot.invalid_count)
print(snapshot.invalid_row_labels)
print(snapshot.visible_rows)
print(snapshot.structured_rows)
필드설명
visible_rows현재 화면에서 보이는 row label 목록입니다.
structured_rowsrow label, editable, disabled, controlType, options를 포함한 구조화 row 상태입니다.
invalid_count저장 후 표시되는 query/invalid 표시 개수입니다.
invalid_row_labelsquery가 표시된 row label 목록입니다.

Row-State Guard

생성 notebook은 다음과 같은 패턴으로 비활성 필드를 자동으로 건너뜁니다.

if row_is_editable("비가임 사유"):
    agent.select_radio("비가임 사유", "기타", page_id="DM", visit_id="V1")
else:
    print("SKIP not editable:", "비가임 사유")

따라서 가임 여부 = 예 선택 후 비가임 사유가 비활성화되는 화면에서는 해당 입력에서 멈추지 않고 다음 입력인 수유 여부로 넘어갈 수 있습니다.

주요 API

연결

ping(), clients(), inspect()

입력

set_date(), set_text(), select_radio(), select_option()

저장/이동

click_save(), click_save_next(), go_to_page(), navigate_to()

Query 확인

has_query(), check_result("Query"), check_result("No Query")

주의사항

  • Query 발생 여부는 입력 직후가 아니라 저장 후 확인합니다.
  • 현재 페이지에 이미 있으면 notebook은 불필요한 이동을 피할 수 있습니다.
  • extension runner JS 수정 후에는 Chrome extension 또는 Tampermonkey script를 다시 로드해야 새 inspect 필드가 반영됩니다.
